WCAG 2.0 POUR Principles in Detail

Perceivable

  • Text Alternatives: <img alt="Description of image">
  • Multimedia Captions: <track kind="captions" src="captions.vtt">
  • Adaptable Content: Use responsive design and relative units
  • Contrast Ratio: Minimum 4.5:1 for normal text

Operable

  • Keyboard Access: tabindex="0" for custom controls
  • Timing: Provide pause/stop controls
  • Seizure Prevention: No content flashing more than 3 times/second
  • Navigation: Clear headings and landmarks

Understandable

  • Readable Text: Use clear fonts and proper spacing
  • Predictable: Consistent navigation and functionality
  • Input Assistance: Clear labels and error messages

Robust

  • Compatibility: Valid HTML and ARIA usage
  • Future-proof: Follow web standards

Semantic HTML Implementation

Document Structure

<header role="banner">
    <h1>Page Title</h1>
</header>

<nav role="navigation">
    <!-- Navigation content -->
</nav>

<main role="main">
    <section>
        <h2>Section Title</h2>
        <article>
            <h3>Article Title</h3>
        </article>
    </section>
    
    <aside>
        <h2>Related Content</h2>
    </aside>
</main>

<footer role="contentinfo">
    <!-- Footer content -->
</footer>

Heading Hierarchy

  • <h1> - Main page title
  • <h2> - Major sections
  • <h3> - Subsections
  • <h4> - Sub-subsections
  • Never skip heading levels

ARIA Implementation Rules

Best Practices

  • Use native HTML elements when possible
  • Only add ARIA when HTML semantics aren't sufficient
  • Ensure all interactive elements are keyboard-accessible
  • Always provide accessible names for interactive elements

Example Implementation

<!-- Good: Native button -->
<button type="button">Click Me</button>

<!-- Avoid: DIV as button -->
<div role="button" 
     tabindex="0" 
     onclick="handleClick()"
     onkeydown="handleKeydown()">
    Click Me
</div>

CSS Layout Techniques

Grid Layout

.page-layout {
    display: grid;
    grid-template-areas:
        "header header"
        "nav main"
        "footer footer";
    gap: 1rem;
}

Flexbox Components

.card-container {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
    justify-content: space-between;
}

.card {
    flex: 1 1 300px;
}

Mobile-First Design

/* Base mobile styles */
.container {
    width: 100%;
    padding: 1rem;
}

/* Tablet and up */
@media (min-width: 768px) {
    .container {
        max-width: 750px;
        margin: 0 auto;
    }
}

Creating an Accessible Website: Implementation Guide

Accessible Forms

Best Practice Example:

<form role="form" aria-label="Contact Form">
    <div class="form-group">
        <label for="name" id="name-label">
            Name
            <span class="required" aria-hidden="true">*</span>
        </label>
        <input 
            type="text" 
            id="name" 
            name="name" 
            required 
            aria-required="true"
            aria-describedby="name-help">
        <span id="name-help" class="help-text">
            Enter your full name
        </span>
    </div>
</form>

Image Accessibility

  • Informative Images: <img src="chart.png" alt="Sales chart showing 50% increase in Q4">
  • Decorative Images: <img src="divider.png" alt="" role="presentation">
  • Complex Images: <figure> <img src="data.png" alt="Quarterly sales data"> <figcaption>Detailed description of the data visualization</figcaption> </figure>

Focus Management

/* Visible focus indicators */
:focus {
    outline: 3px solid #4A90E2;
    outline-offset: 2px;
}

/* Skip Link Implementation */
.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    padding: 8px;
    background: #4A90E2;
    color: white;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}

Color and Contrast Guidelines

  • Normal text: 4.5:1 contrast ratio
  • Large text (18pt+): 3:1 contrast ratio
  • Don't rely on color alone for information
  • Provide patterns or icons with colors

  • ✓ Check color contrast
    • WebAIM Contrast Checker
    • Chrome DevTools
  • ✓ Validate HTML structure
  • ✓ Test responsive behavior

Accessible Interactive Patterns

<!-- Modal Dialog -->
<dialog 
    role="dialog"
    aria-labelledby="dialog-title"
    aria-describedby="dialog-desc">
    <h2 id="dialog-title">Dialog Title</h2>
    <p id="dialog-desc">Description here</p>
    <button autofocus>Close</button>
</dialog>

<!-- Dropdown Menu -->
<div class="dropdown">
    <button 
        aria-haspopup="true"
        aria-expanded="false">
        Menu
    </button>
    <ul role="menu">
        <li role="menuitem">Option 1</li>
    </ul>
</div>
