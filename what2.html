Norman's Design Concepts in Detail

Affordance

Affordance refers to the perceived and actual properties of an object that suggest how it should be used. For example:

  • Buttons with a raised appearance suggest they can be pressed
  • Underlined text suggests it's clickable
  • A slider control suggests it can be dragged
  • Input fields with a white background suggest they can be typed in

Constraints

Constraints are design elements that limit possible actions to prevent errors and guide users. Examples include:

  • Graying out unavailable options in a menu
  • Date pickers that prevent invalid date selections
  • Character limits on input fields
  • Form validation that prevents submission of incorrect data

Feedback

Feedback provides immediate response to user actions, confirming their interaction. Common implementations:

  • Button color changes when clicked
  • Success messages after form submission
  • Error messages when something goes wrong
  • Loading indicators during processes
  • Sound effects for important notifications

Visibility

Visibility ensures users can easily find and understand available functions. Key aspects include:

  • Clear navigation menus
  • Obvious call-to-action buttons
  • Important information displayed prominently
  • Status indicators (e.g., progress bars)
  • Clear hierarchy of information

Mapping

Mapping creates natural relationships between controls and their effects. Examples include:

  • Volume slider that moves up for higher volume
  • Left/right arrows for previous/next navigation
  • Spatial layout matching real-world relationships
  • Icons that visually represent their function

Typography Best Practices

Text Contrast

Ensuring readability through proper contrast:

  • Dark text (#333333) on light backgrounds
  • Avoid pure black on pure white
  • Test contrast with accessibility tools
  • Maintain contrast in all color schemes

Line Specifications

Optimal spacing and lengths:

  • Line height: 1.5 to 1.6 times font size
  • Ideal line length: 50-75 characters
  • Paragraph spacing: 1.5em
  • Margins prevent text touching edges

Font Selection

Choosing the right fonts:

  • Sans-serif for body text (Open Sans, Roboto)
  • Serif for headings (Merriweather, Georgia)
  • Maximum 2-3 font families per design
  • Include web-safe font fallbacks

Size Guidelines

Maintaining readable sizes:

  • Body text: minimum 16px
  • Mobile text: minimum 18px
  • Headings: 20% larger than body minimum
  • Consistent size hierarchy

Color Design Requirements

Normal Text Contrast

WCAG 2.1 requirements for standard text:

  • 4.5:1 minimum contrast ratio
  • Applies to text under 18pt
  • Includes form inputs
  • Use contrast checker tools

Large Text Contrast

Requirements for larger text elements:

  • 3:1 minimum contrast ratio
  • Applies to text over 18pt
  • Or 14pt if bold
  • Includes large navigation items

Color Blindness

Ensuring color-blind accessibility:

  • Use colorblind-safe palettes
  • Avoid red/green combinations
  • Test with color blindness simulators
  • Consider monochrome users

Information Conveying

Multiple ways to convey information:

  • Never rely on color alone
  • Add icons or patterns
  • Include text labels
  • Provide alternative indicators

Testing Methodology

HTML Validation

Using W3C HTML Validator:

  • Check document structure
  • Verify required attributes
  • Ensure proper nesting
  • Validate semantic markup

CSS Validation

Style sheet verification:

  • Check syntax validity
  • Verify browser compatibility
  • Identify unused styles
  • Validate media queries

WAVE Testing

Accessibility evaluation:

  • Check contrast ratios
  • Verify alt text presence
  • Review ARIA usage
  • Identify structural issues

Lighthouse Audits

Performance and accessibility checks:

  • Run performance tests
  • Check accessibility scores
  • Review best practices
  • Verify SEO optimization

Screen Readers

Testing with multiple tools:

  • JAWS for Windows testing
  • VoiceOver for Mac testing
  • NVDA for free testing
  • Check reading order

Keyboard Navigation

Testing keyboard accessibility:

  • Verify logical tab order
  • Test skip navigation
  • Check focus indicators
  • Ensure no keyboard traps
